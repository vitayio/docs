---
title: "Question sets and templates"
description: "Create and manage the question sets that references answer, and use templates to speed up setup."
---

A **question set** is the list of questions Vitay sends to each reference when a reference request is sent. You can have multiple question sets — for example, one for executive roles and one for entry-level roles — and select the right one when you create a reference request.

<Note>
  You need a **root recruiter** or **supervisor recruiter** role to create, edit, or archive question sets. Regular recruiters can select from existing question sets but cannot modify them.
</Note>

## Create a question set

<Steps>
  <Step title="Open Question sets">
    Go to **Settings → Question sets** (sometimes listed as **Question Sets** in the navigation).

    ![SCREENSHOT: The Question sets list page with an "Add" or "Create" button — placeholder](/images/placeholder.png)
  </Step>
  <Step title="Click Add (or Create)">
    Click the **+ Add** button. A new question set form opens.
  </Step>
  <Step title="Give it a title">
    Enter a clear, descriptive **title** so recruiters can identify it in the dropdown when sending a reference request (for example, "Standard Reference Check" or "Healthcare — Clinical Roles").
  </Step>
  <Step title="Choose a language">
    Select the **language** references will answer in. If your plan includes multiple languages, you can create separate question sets per language.
  </Step>
  <Step title="Add questions">
    Click **Add question** and choose a question type:

    | Type | When to use |
    |---|---|
    | **Text** | Open-ended written answer |
    | **Rating** | Numeric score on a scale (e.g., 1–5) |
    | **Multiple choice (Options)** | Pick one from a list |
    | **Radio** | Single-select from options displayed as radio buttons |
    | **Checkbox** | Multi-select from a list |
    | **Matrix** | Rate several items on the same scale in a grid |
    | **Slider** | Drag to a value on a continuous scale |
    | **Ranking** | Drag items into a ranked order |
    | **Date** | Pick a single date |
    | **Date range** | Pick a start and end date |

    For each question, enter the question text and — for choice-based types — add the answer options. You can also mark a question as **required** and, for rating/text questions, allow an additional comment field.
  </Step>
  <Step title="Organise questions into groups (optional)">
    If you have created **question groups** (labels that organise questions into themed sections), assign each question to a group. Groups appear as section headings in the reference form.
  </Step>
  <Step title="Save">
    Click **Save**. The question set is now available to select when sending a reference request.

    ![SCREENSHOT: A completed question set with several questions and groups — placeholder](/images/placeholder.png)
  </Step>
</Steps>

## Edit a question set

<Steps>
  <Step title="Find the question set">
    Go to **Settings → Question sets** and click the set you want to change.
  </Step>
  <Step title="Make your edits">
    Add, remove, or reorder questions. You can drag questions to change their order. Changes only affect future reference requests — submitted answers are not altered.
  </Step>
  <Step title="Save">
    Click **Save**.
  </Step>
</Steps>

## Set a default question set

If you have only one active question set, Vitay selects it automatically. If you have multiple, you can mark one as the **default** so it is pre-selected when a recruiter creates a reference request.

Open the question set and look for a **Set as default** option (or a "Default" toggle). Only one question set can be the default at a time.

## Archive a question set

Archiving hides a question set from the selection dropdown without deleting it or its historical data.

<Steps>
  <Step title="Find the question set">
    Go to **Settings → Question sets**.
  </Step>
  <Step title="Deactivate it">
    Use the **archive** or **deactivate** control next to the question set. It will no longer appear when recruiters send new reference requests, but all past references linked to it remain intact.
  </Step>
</Steps>

## Use question set templates

If your plan includes **question set templates** (check **Settings → Question set templates**), Vitay provides pre-built question sets you can use as a starting point.

<Steps>
  <Step title="Open Question set templates">
    Go to **Settings → Question set templates**.
  </Step>
  <Step title="Browse available templates">
    Each template has a title and a language. Templates are read-only — you copy them rather than edit them directly.
  </Step>
  <Step title="Copy a template to a question set">
    Click **Use template** (or a similar copy action). Vitay creates a new question set pre-populated with the template's questions. You can then edit that copy however you need.

    ![SCREENSHOT: The question set templates list with a "Use template" button on each row — placeholder](/images/placeholder.png)
  </Step>
</Steps>

<Note>
  Question set templates are managed by Vitay. If you need a template added or modified, contact **support@vitay.io**.
</Note>

## Question groups

**Question groups** are labels that let you organise questions under thematic headings (for example, "Leadership" or "Technical skills"). Groups appear as section headers in the reference form and make longer question sets easier to navigate.

To manage groups, go to **Settings → Question groups**. You can add, rename, or remove groups there. Groups are shared across all question sets on your account.

## Troubleshooting

<AccordionGroup>
  <Accordion title="I can't select a question set when sending a reference request">
    There may be no active question sets, or all sets may have been archived. Go to **Settings → Question sets** and check that at least one set has an active status. If the list is empty, create a new set following the steps above.
  </Accordion>
  <Accordion title="The Question set templates section is not visible">
    Templates are a feature that must be enabled for your account. If you don't see the section, contact **support@vitay.io** to enquire about enabling it.
  </Accordion>
  <Accordion title="I want different question sets for different relationship types">
    You can assign a question set to a specific reference relationship (e.g., "Direct manager" always gets the Manager question set). Go to **Settings → Reference relationships** and select a default question set for each relationship type there.
  </Accordion>
</AccordionGroup>
